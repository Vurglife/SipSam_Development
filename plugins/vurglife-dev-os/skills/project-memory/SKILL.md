Use `docs/system-development` as the current project memory.

Read only the files needed for the task:

- `BRIEF.md`: purpose, users, scope, constraints, success criteria.
- `DESIGN.md`: architecture, UX, AI/automation, data design, confirmed patterns.
- `DECISIONS.md`: durable decisions and revisit triggers.
- `VALIDATION.md`: checks performed, evidence, gaps.
- `LESSONS.md`: reusable patterns and failed approaches not to repeat.
- `ROADMAP.md`: now, next, later, risks, deferred work.

When meaningful work changes the project, update the smallest relevant file. Keep entries concise, dated, and evidence-based.

Do not paste large documents into chat. Store durable detail in project files and summarize only what matters.
